01 / Problem

The Problem

Digital tickets and access passes lack any trustless verification layer. The status quo is fragile.

  • 📸

    Screenshots can be faked

    Anyone can capture or forge a ticket image. There is no way to distinguish a real credential from a counterfeit one.

  • 🏛️

    Centralized systems are single points of failure

    Backend outages, database breaches, or vendor lock-in can invalidate every credential at once.

  • 🔓

    No trustless, offline-capable verification exists

    Existing solutions require internet connectivity and a trusted authority — the very things Web3 eliminates.

02 / Solution
Our Solution
ScanProof lets anyone issue and verify cryptographic QR proofs signed by a real Solana wallet.
No backend needed — the signature is the proof.

Using Mobile Wallet Adapter, any Solana wallet can sign a canonical proof envelope. The resulting QR code is self-contained and verifiable by anyone, anywhere — no server, no authority, no trust assumptions.

03 / How It Works

How It Works

01
Connect Wallet via MWA
Authenticate with any Solana wallet using Mobile Wallet Adapter. Your public key becomes the issuer identity.
→
02
Sign Proof Envelope
The wallet signs a canonical JSON payload with your private key. SHA-256 hashing ensures tamper evidence.
→
03
Share QR Code
The signed envelope is base58-encoded and displayed as a portable, shareable QR code.
→
04
Scan & Verify Instantly
Any ScanProof user scans the QR. The signature is verified against the public key. No internet required.

06 / Security Model

Security Model

Every design decision in ScanProof is rooted in cryptographic first principles — not convenience shortcuts.

📋

Canonical JSON Serialisation

Proof payloads are deterministically serialised before hashing to prevent trivial collision attacks.

#

SHA-256 Payload Hashing

A content hash is embedded in every envelope, providing tamper evidence independent of the signature.

✂

Detached Signature Verification

Signatures are verified against the claimed issuer's public key using TweetNaCl ed25519.

⏱

Time-Window Expiration Checks

Proof envelopes carry issuance and expiry timestamps; verifiers reject expired credentials automatically.

🔑

No Central Authority — Wallet Key Is the Trust Anchor

There is no server to hack, no database to breach, no vendor to trust. The Solana wallet keypair is the sole root of trust. Verification requires only the issuer's public key.
